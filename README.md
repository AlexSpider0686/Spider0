# Smeta App Site + System

Публичный сайт и внутренняя система расчета в одном проекте:

- `/` — маркетинговый лендинг
- `/legal/privacy` — политика конфиденциальности
- `/legal/personal-data` — политика обработки персональных данных
- `/legal/cookies` — политика cookies
- `/legal/disclaimer` — отказ от ответственности по точности расчетов
- `/system` — калькулятор предварительной бюджетной оценки

## Что реализовано

- видеоподложка на лендинге (с fallback постером)
- вход в систему через email + тестовый 6-значный код `123456`
- обязательный чекбокс согласия перед входом
- cookies banner с режимами согласия
- legal-страницы в структуре, пригодной для GitHub/Vercel

## Локальный запуск

```bash
npm install
npm run dev
```

## Сборка

```bash
npm run build
```

## Важные production-шаги перед релизом

1. Заполнить реальные реквизиты оператора ПДн (юрлицо, адрес, контакты).
2. Включить реальную доставку OTP-кода по email и отключить тестовый режим.
3. Добавить лицензированный loop-видеофайл в `public/media/manhattan-loop.mp4` (3-4 минуты).
4. Финально проверить тексты legal-документов под фактические процессы компании.
